# TODO Manager

管理用户的待办事项列表。

## TODO 文件位置

```
/home/admin/.openclaw/workspace/todo.md
```

## 命令格式

| 命令 | 作用 | 示例 |
|------|------|------|
| `+ 任务内容` | 添加新任务 | `+ 写完周报` |
| `完成 N` | 标记第 N 个任务为完成 | `完成 3` |
| 待办/todo/任务 | 查看当前列表 | `待办` / `todo` |

## 工作流程

### 查看任务

当用户提到「待办」「todo」「任务」「要做的事」等关键词时：
1. 读取 `todo.md`
2. 返回简洁的任务列表（只显示待办事项，不显示已完成）

### 添加任务

当用户发送 `+ 任务内容` 时：
1. 读取 `todo.md`
2. 在待办事项列表末尾添加新任务
3. 更新时间戳
4. 回复「✅ 已添加：任务内容」

### 完成任务

当用户发送 `完成 N` 时：
1. 读取 `todo.md`
2. 找到第 N 个待办事项
3. 将其移到「已完成」区域
4. 更新时间戳
5. 回复「✅ 已完成：任务内容」

## 文件格式

```markdown
# 📋 TODO List

> 更新时间：YYYY-MM-DD HH:MM

## 待办事项

1. [ ] 任务一
2. [ ] 任务二
3. [ ] 任务三

## 已完成

- [x] 已完成的任务 ✓

---
**交互命令：**
- `+ 任务内容` → 添加新任务
- `完成 N` → 标记第 N 个任务为完成
```

## 注意事项

- 任务编号从 1 开始
- 已完成的任务移到「已完成」区域，带 ✓ 标记
- 每次操作后更新时间戳
- 回复简洁，不重复整个列表（除非用户要求）
